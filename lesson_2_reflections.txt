Working directory show the changes made not waiting to commit.
The staging area represents changes pending to commit.
The repository are changes committed.

This system makes it easier before commiting any changes to differiante very well the changes.

How can you use the staging area to make sure you have one commit per logical change?

By checking the changes made to the code before committing using the command git diff --staged
